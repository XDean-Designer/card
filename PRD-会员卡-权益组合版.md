## 1. 背景与目标

### 1.1 背景

门店当前建卡能力常按「卡类型」配置（储值、次卡、折扣等），规则分散，有效期多层、退卡与计绩口径不统一。原型以 **权益组合** 统一：**面值 / 项目 / 折扣** 可组合，并用 **卡级有效期 + 用卡策略（均值/减值）** 统一延期、退卡前提。APP 侧均为当前建卡体系，无独立「旧卡产品线」需替换。

### 1.2 当前排期目标（仅会员卡）

1. 商家能在手机端完成：**创建组合卡模板 → 办卡 / 延期 / 整卡退卡**。
2. 模板治理清晰：**有持卡时规则锁定；下架 / 复制建卡 / 重新上架**可运营。
3. 退卡给出可解释的 **建议退款**，支持店员改 **实际退款** 并落 **审计日志**。
4. Step2 **计次模式（每项/共计）**、Step3 **双轴策略**与详情 **运营数据 / 卡面信息** 布局与原型一致。

### 1.3 非目标（当前排期明确不做）

| 项 | 说明 |
|----|------|
| **项目管理（价目表）全套能力** | **本期不做**；产品边界见 §4，后续单独立项 |
| **会员管理** | **尚未启动**；办卡「选会员」暂依赖既存会员能力/接口，见 §5 |
| 劳动业绩 runtime | 仅持久化 `performanceRule` + Step3 文案；**无核销计绩运算** |
| 自建核销 / 服务单 | 后续；本期消耗数据来自既存核销或持卡账本快照 |
| 退卡手续费、超额审批、退卡备注 | 本期不做；可改实退须审计；高于建议 **无拦截** |
| 部分退 | **产品决策：不支持**（仅整卡退） |
| 适用对象 / 卡项说明 | UI 已移除，不做 |
| **补录老卡** | **本期不做；P1 新增**（见 §6.8 · §9） |
| 顾客端预约真联调 | 后续 |
| 权益定制版等姊妹线 | 不入本期排期 |

### 1.4 数据依赖（本期不建设、但流程需要）

| 依赖 | 本期策略 |
|------|----------|
| 可选项目 / 挂牌价 | **不建设价目管理**；项目数据暂用 **既存源或占位数据**，不纳入本期交付与验收 |
| 会员档案 | **会员管理未启动**；办卡选会员对接 **既存会员接口/列表**；会员 CRUD 属后续「会员管理」 |

### 1.5 成功标准（评审用）

- 「会员卡创建与管理」主路径可走通，退卡算例与附录公式一致（至少算例 A）。
- 计次模式（每项/共计）、纯折扣/不限次退卡分支与原型引擎一致。
- 项目管理、会员管理不进入本期验收口。
- 规则冲突时以 **逻辑文档 + 引擎** 为准（非界面简化文案）。

## 6. 当前排期 · 会员卡创建与管理

### 6.1 能力范围（本期交付）

| 能力 | 本期 |
|------|------|
| 列表（在售/已下架）、详情、运营数据展示 | ✅ |
| 三步创建（基本信息 → 权益组合 → 用卡策略） | ✅ |
| 计次模式（每项/共计）、不限次 × 永久拦截 | ✅ |
| 模板锁定 / 复制建卡 / 下架 / 重新上架 | ✅ |
| 办卡、延期、整卡退卡估值与确认 | ✅ |
| `performanceRule` 字段存储 + Step3 文案 | ✅（无运算） |
| 实际退款可改 + 审计日志 | ✅ |
| 劳动业绩运算、自建核销 | ❌ 后续 |
| 项目管理 / 会员管理 | ❌ 见 §4 · §5 |
| 补录老卡 | ❌ 本期；**P1** 见 §6.8 · §9 |

### 6.2 对象模型（逻辑）

```text
卡模板 Template
  name, purchaseAmount / recharge, cardValidity, cardTheme
  benefits: { balance?, timesOrValidity?, projectDiscount? }
  projectItems[]: name, purchaseQty, giftQty, unlimited,
                  qtyScope (per_item | shared), sharedGroupId?
  memberPrices{}: 折扣规则（discount tick / fixed amount）
  usagePolicy: { refundRule, performanceRule }   // alloc_pool | retail
  shelved, createdAt, duplicatedFrom?

持卡实例 CardInstance（办卡快照）
  memberId, templateIdSnapshot, paidAmount
  issuedAt, cardExpireAt
  ledger: balance / projects / discountUses(savedTotal)
  usagePolicySnapshot
  catalogPriceSnapshot
  status: active | expired | refunded
```

**账本 · 共计组**：同 `sharedGroupId` 内仅 **primary** 项目写入完整剩余次数，其余项目 `remaining=0` 可记 `used`；退卡时汇总组内已用/剩余。

### 6.3 创建三步

#### Step1 · 基本信息

| 字段 | 规则 |
|------|------|
| 卡名称 * | 必填 |
| 购买金额 * | 必填，>0；办卡实付基准 |
| 有效期限 * | 日 / 月 / 年 / **永久**（默认永久）；**仅约束项目与折扣** |
| 卡面模板 | 主题色可选 + 预览 |

已移除：适用对象、卡项说明。

#### Step2 · 权益组合（至少勾选一类）

| 权益 | 配置 |
|------|------|
| 面值 | 面值金额、赠送金额；**永久有效**（不受卡到期约束） |
| 项目 | 选择项目（展示已选 N 项）→ **计次模式** + **次数设置** |
| 折扣 | 选择项目 → 标尺折扣（`tickIndex` 0–20）或固定会员价 |

**计次模式（`qtyScope`）**

| 模式 | 枚举 | 规则 |
|------|------|------|
| **每项** | `per_item`（默认） | 每个选中项目**各自**获得相同次数；不限次时每个项目独立不限次 |
| **共计** | `shared` | 所选项目**任意混用**，共享同一次数池；同组 `sharedGroupId`（= 权益卡 id） |
| 不限次约束 | — | 强制 `per_item`；「共计」单选禁用，右侧 `?` 仍可点 |

UI：计次模式「每项 / 共计」**横向排列**；各选项旁圆圈 `?` 打开说明弹窗（展示当前状态文案）。

**次数设置**：权益次数 / 赠送次数 / 不限次（与步进器互斥）。文案已定稿：原「购买次数」→ 权益次数；原「次数范围」→ 计次模式；原「次数模式」→ 次数设置。

**不限次 × 永久**

- 选「不限次」且卡有效期为永久：居中弹窗「**设置有效期限**」；正文说明不限次不可用永久；内嵌日/月/年（默认 1 年）。
- **取消** → 不保留不限次；**确定** → 写回卡有效期并保留不限次。
- 已有不限次时不可改回永久（toast：「已有不限次项目，卡有效期不可为永久」）。

其他交互：权益 Switch 关闭仅收起，**保留已配数据**；结果卡展示项目范围（全选 →「全部项目」）与次数/折扣摘要；添加权益确定后对应区块保持展开。

#### Step3 · 用卡策略（必填）

```text
usagePolicy: {
  refundRule:      alloc_pool | retail   // 界面：均值模式 / 减值模式；默认均值
  performanceRule: alloc_pool | retail   // 界面：均值模式 / 原价模式；默认均值
}
```

- 两轴 **独立**，共 4 种组合；办卡写入实例快照。
- 分区标题定稿：**消耗及退卡计算规则**、**业绩计算规则**。
- 纯折扣 / 不限次消耗副标题写「**总有效期**」，**不**直接展示具体天数；明细见 `?` 弹窗（永久纯折扣弹窗内写 1095 天）。
- 纯折扣 · 均值业绩文案：**员工业绩 = 项目折后价格**。
- 不限次独占 · 均值业绩文案：价目表原价。
- **本期只存字段 + 展示文案，不实现计绩 runtime**。

### 6.4 模板锁定与复制建卡

| 条件 | 行为 |
|------|------|
| 当前有效持卡数 > 0 | Step1–3 **不可编辑**；可下架/上架、**复制建卡**、办/延/退 |
| 当前有效 = 0 | 可编辑 |
| 需改规则且仍有持卡 | **复制建卡**为新模板；**不强制**下架旧模板 |

- 锁定提示：「已有持卡，不可编辑；请复制建卡后修改」。
- 「当前有效」= 未退卡且卡级未过期；已退卡 / 已失效 **不计入**锁定。
- 复制建卡：新卡名默认 `{原名}（副本）`；三步顶栏说明「正在基于「原卡」创建新卡，保存后原卡不受影响」；新卡默认上架；可记 `duplicatedFrom`。
- 下架：不可新办；已持卡仍可退卡/延期。已下架详情顶栏「已下架 + 复制建卡」；底栏「重新上架 / 已持卡管理」。

### 6.5 办卡

- 入口：列表卡面「办卡/退卡」、详情、创建成功页。
- 选择会员（既存会员能力）→ 支付 → 生成实例：策略快照、挂牌价快照、初始 ledger、到期日（非永久）。
- 面值不因卡到期而清除。

### 6.6 延期

- 仅延长 **卡到期日**（`cardExpireAt += N` 或延为永久）；已失效可延期复活。
- **永久卡 / 纯面值卡**：仍展示延期入口；点击 toast「**该卡永久有效，无需延期**」，不展开面板；接口拒绝。
- **无折扣**：免费延期。
- **含折扣**：可收费；默认参考费用 = `办卡实付 × 延期天数 ÷ 原有效期`；商家可改或填 0；独立流水，**不计入办卡总额/次数**。
- **不限次 × 永久**：建卡时弹窗设有效期（见 §6.3）；非延期流程本身。

### 6.7 退卡

- **整卡退**；建议退款见 §7.4；**不以 Σ剩余封顶**。
- Sheet：剩余上、消耗下；策略角标「均值退卡 / 减值退卡」；文案「**建议退款**」。
- 可改实际退款 + **审计日志**；无手续费 / 审批 / 备注；高于建议无拦截。
- 消耗：既存核销汇总或账本快照。
- 确认后实例退卡、权益清零。

### 6.8 补录老卡（P1，非本期）

- APP 内**不存在另一套「旧卡产品线」**；补录是指将**线下或未进系统的持卡**录入 **当前建卡体系**（选模板 / 权益 / 原办卡日等）。
- **本期不交付**；**P1 新增**。细则可对齐 `logic.html`：必填原办卡日，`卡到期日 = 原办卡日 + 模板有效期`（永久模板则到期为空）。
- 补录统计口径（是否计入办卡次数/总额）P1 立项时另定。

### 6.9 详情页布局

```text
卡头（独立彩色卡条：名称 · 状态 · 复制建卡 / 下架 / 编辑）

【运营数据】（白底板块 · 标题加粗 + 释义 ?）
  办卡总额 | 办卡次数 | 办卡人数
  持卡余额（有面值时）| 当前有效
  各项目消费次数 / 剩余次数

【卡面信息】（白底板块 · 板内细分割）
```

### 6.10 主路径

```text
列表 → 添加会员卡 → Step1 → Step2 → Step3 → 保存
     → 办卡 → 选会员 → 支付成功

列表 → 下架 / 重新上架 / 复制建卡
详情 → 有持卡时编辑禁用 → 复制建卡改规则
```

### 6.11 验收要点（仅会员卡 · 本期）

### 7.1 Q1 · 有效期

|------|------|
| 卡级有效期 | 仅约束 **项目、折扣**；办卡写入 `cardExpireAt` |
| 面值 | **永久有效**；过期后仍可消费 / 参与退卡 |
| 宽限期 | **无**；到期日当日项目/折扣硬失效 |
| 恢复 | 延期复活或重新办卡 |
| 不限次 | 卡级有效期 **不得为永久**（见 §6.3） |
| 单项有效期 | **已删除**；项目/折扣不再有独立有效期配置 |

### 7.2 Q2 · 延期

见 §6.6。延长卡有效期 = 延长全部项目/折扣可用期。

### 7.3 Q3 · 劳动业绩（产品目标 · 本期不实现运算）

- 字段 `performanceRule` 本期必存（界面：均值 / **原价**）。
- Step3 文案：
  - 组合卡均值：员工业绩 = 项目面值 ×（办卡实付 ÷ 总权益面值）
  - 纯折扣均值：员工业绩 = **项目折后价格**
  - 不限次独占均值：价目表原价
  - 原价模式：员工业绩 = 项目面值（挂牌价）
- 公式细节见 `logic.html` Q3；**不得写入本期验收为运算结果**。
- 退卡只读 `refundRule`，与计绩不强制共用 runtime。

### 7.4 Q4 · 退卡公式

**建议退款（均值 / 减值共通）**

```text
建议退款 = max(0, 办卡实付 − Σ消耗价值)
```

- Σ剩余价值可展示，**不参与封顶**。
- 消费顺序：余额先充值后赠送；项目先购买次后赠送次。
- Sheet 用语统一为 **「消耗价值」**。

#### 名义价值池（均值分母 · 总权益面值）

| 权重 | 定义 |
|------|------|
| 余额名义 | 充值 + 赠送面值 |
| 项目权重（每项） | 挂牌价 × (购次+赠次)；不限次不参与；各项目各计一条 |
| 项目权重（共计） | **组内挂牌价均价 × (购次+赠次)**；同 `sharedGroupId` **整组只计一条** |
| 权重合计 | 面值 + 有限次项目权重（**不含折扣、不含不限次**） |

组合卡折扣线：消耗 = 累计已享优惠（`savedTotal`），**计入** Σ消耗；**不参与**分摊分母（剩余明细 = 0）。

#### 均值 · 消耗价值（`alloc_pool`）

| 线 | 消耗价值 |
|----|----------|
| 余额 | 已用充值/赠送按分摊实付折算（充值、赠送分系数） |
| 项目（每项） | 已用 × (分摊实付 ÷ 总次数) |
| 项目（共计） | 组内已用合计 × (组分摊实付 ÷ 总次数)；Sheet 标签「共计 · …」 |
| 折扣（组合卡） | 累计已享优惠；不参与分摊分母，剩余 = 0 |
| **纯折扣卡** | 办卡实付 ×（已过天数 ÷ **总有效期**）；永久按 **1095** 天；已享优惠仅信息行，**不计入消耗** |
| 固定期限不限次独占 | 无面值/有限次权重时同时间摊销；组合卡实付已由面值/有限次分完时**不再另占份额** |

时间摊销 **仅** 均值模式。

#### 减值 · 消耗价值（`retail`）

| 线 | 消耗价值 |
|----|----------|
| 余额 | 同均值 |
| 项目（每项） | **挂牌价 × 已用**（办卡快照价） |
| 项目（共计） | **组内均价 × 组内已用合计** |
| 折扣（组合/纯折扣） | 累计已享优惠；剩余 = 0 |
| **纯折扣卡** | **无**时间摊销；仅 `savedTotal`（对照演示「减值烫染卡」） |
| 不限次独占 | **无**时间摊销；仅按余额/项目/折扣账本 |

界面「按挂牌价」仅精确作用于 **项目线**；折扣与均值同为 `savedTotal`。

#### 折扣档换算

- `tickIndex` ∈ [0, 20]；`20` = 原价；折后价 = 原价 × `tickIndex / 20`
- `mode=fixed`：折后价 = 配置金额
- 已享优惠单次 = max(0, 原价 − 折后价)；累计 `savedTotal`

#### 锚定算例 A · 纯次卡

实付 **500**；洗剪吹挂牌价 **68**；购 **8** + 赠 **2**；已用购买 **2** 次。

| 模式 | 计算 | 建议退 |
|------|------|--------|
| 均值 | 单次摊销 50 × 2 = 100 | **400** |
| 减值 | 68 × 2 = 136 | **364** |

更多算例见 `logic.html` B–E。

### 7.5 策略默认与冻结

- 默认两端均为 `alloc_pool`（均值）。
- 办卡后策略与挂牌价快照冻结；改模板 / 日后改价 **不回溯**已办卡。
