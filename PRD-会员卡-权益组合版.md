## 1. 背景与目标

### 1.1 背景

门店当前建卡能力常按「卡类型」配置（储值、次卡、折扣等），规则分散，有效期多层、退卡与计绩口径不统一。原型以 **权益组合** 统一：**面值 / 项目 / 折扣** 可组合，并用 **卡级有效期 + 用卡策略（均值/减值）** 统一延期、退卡前提。APP 侧均为当前建卡体系，无独立「旧卡产品线」需替换。

### 1.2 当前排期目标（仅会员卡）

1. 商家能在手机端完成：**创建组合卡模板 → 办卡 / 延期 / 整卡退卡**。
2. 模板治理清晰：**持卡锁定不可乱改规则；下架/克隆/重新上架**可运营。
3. 退卡给出可解释的 **建议退款**，支持店员改 **实际退款** 并落 **审计日志**。

### 1.3 非目标（当前排期明确不做）

| 项 | 说明 |
|----|------|
| **项目管理（价目表）全套能力** | **本期不做**；产品边界见 §4，后续单独立项 |
| **会员管理** | **尚未启动**；办卡「选会员」暂依赖既存会员能力/接口，见 §5 |
| 劳动业绩 runtime | 仅持久化 `performanceRule` + Step3 文案；运算后续 / Q3 |
| 自建核销 / 服务单 | 后续；本期消耗数据来自既存核销或持卡账本快照 |
| 退卡手续费、超额审批、退卡备注 | 本期不做；可改实退须审计 |
| 部分退 | **产品决策：不支持** |
| 适用对象 | 不做 |
| **补录老卡** | **本期不做；P1 新增**（见 §9） |
| 顾客端预约真联调 | 后续 |
| 权益定制版等姊妹线 | 不入本期排期 |

### 1.4 数据依赖（本期不建设、但流程需要）

| 依赖 | 本期策略 |
|------|----------|
| 可选项目 / 挂牌价 | **不建设价目管理**；项目数据暂用 **既存源或占位数据**，不纳入本期交付与验收 |
| 会员档案 | **会员管理未启动**；办卡选会员对接 **既存会员接口/列表**；会员 CRUD 属后续「会员管理」 |

### 1.5 成功标准（评审用）

- 「会员卡创建与管理」主路径可走通，退卡算例与附录公式一致。
- 项目管理、会员管理不进入本期验收口。
- 规则冲突时以 **逻辑文档 + 引擎** 为准（非界面简化文案）。

## 4. 后续规划 · 项目管理（价目表）— 本期不做

> 与会员卡 **分篇**。选项目、名义价值池挂牌价逻辑上依赖本篇数据；**当前排期不交付本篇任何能力**，仅保留边界与规则预留，避免后续返工。

### 4.1 产品边界（预留）

| 项目管理负责 | 会员卡负责 |
|--------------|------------|
| 门店可售服务目录与挂牌价 | 组合哪些项目进模板、次数/折扣规则 |
| 项目上下架（售卖/预约侧） | 卡模板上下架（可否新办） |
| 不定义退卡公式 | 用挂牌价参与名义池 / 减值消耗 |

### 4.2 字段与绑卡锁定（定稿预留）

| 字段 | 绑卡后 |
|------|--------|
| 名称、在售、手机端预约、置顶、删除 | 不可改 |
| 价格、服务时长、照片 | 可改 |

- **已办卡实例不跟价**（挂牌价以办卡快照为准）。
- 列表字色：普通 / 已绑卡 / 已下架（对齐原型，随本篇上线时交付）。

### 4.3 本期对项目数据的要求

- 卡流程「选择项目」可读到 **名称 + 挂牌价** 即可。
- 数据来源：既存系统或配置占位；**不验收**价目 CRUD、置顶、图例、预约开关联调。

## 6. 当前排期 · 会员卡创建与管理

### 6.1 能力范围（本期交付）

| 能力 | 本期 |
|------|------|
| 列表（在售/已下架）、详情、运营数据展示 | ✅ |
| 三步创建（基本信息 → 权益组合 → 用卡策略） | ✅ |
| 模板锁定 / 克隆 / 下架 / 重新上架 | ✅ |
| 办卡、延期、整卡退卡估值与确认 | ✅ |
| `performanceRule` 字段存储 + Step3 文案 | ✅（无运算） |
| 实际退款可改 + 审计日志 | ✅ |
| 劳动业绩运算、自建核销 | ❌ 后续 |
| 项目管理 / 会员管理 | ❌ 见 §4 · §5 |
| 补录老卡 | ❌ 本期；**P1** 见 §6.8 · §9 |

### 6.2 对象模型（逻辑）

```text
卡模板 Template
  name, purchaseAmount, cardValidity, cardTheme
  benefits: { balance?, projects?, discounts? }
  usagePolicy: { refundRule, performanceRule }   // alloc_pool | retail
  shelved, createdAt, duplicatedFrom?

持卡实例 CardInstance（办卡快照）
  memberId, templateIdSnapshot, paidAmount
  cardExpireAt
  ledger: 余额/次数/折扣已享优惠等
  usagePolicySnapshot
  catalogPriceSnapshot
  status: active | expired | refunded
```

### 6.3 创建三步

**Step1 基本信息**：卡名称 *、购买金额 *、有效期限 *（日/月/年/永久）、卡面模板

**Step2 权益组合**（至少勾选一类）

| 权益 | 配置 |
|------|------|
| 面值 | 面值金额、赠送金额；**永久有效** |
| 项目 | 选择项目（展示已选 N 项）→ 购买/赠送次数或不限次 |
| 折扣 | 选择项目 → 标尺折扣或固定会员价 |

**Step3 用卡策略**（必填）

- **消耗及退卡规则**：均值 / 减值（默认均值）
- **业绩结算规则**：均值 / 减值（默认均值；本期只存不运算）
- 两轴独立；办卡写入实例快照

### 6.4 模板锁定与克隆

| 条件 | 行为 |
|------|------|
| 当前有效持卡数 > 0 | 不可编辑 Step1–3；可下架/上架、克隆、办/延/退 |
| 当前有效 = 0 | 可编辑 |
| 需改规则且仍有持卡 | **克隆**新模板；不强制下架旧模板 |

下架：不可新办；已持卡仍可退卡/延期。

### 6.5 办卡

- 入口：列表卡面「办卡/退卡」、详情、创建成功页
- 选择会员（既存会员能力）→ 支付 → 生成实例与策略/挂牌价快照

### 6.6 延期

- 仅延长 **卡到期日**；已失效可延期复活；永久卡不展示延期
- **含折扣**：延期费用可改（可 0）；独立流水，**不计入办卡总额/次数**
- **无折扣**：免费延期

默认费用（有折扣）：`折扣权益分摊购卡价 × 延期天数 ÷ 原有效期`。

### 6.7 退卡

- **整卡退**；建议退款见附录；**不以 Σ剩余封顶**
- 可改实际退款 + **审计日志**；无手续费/审批/备注
- 消耗：既存核销汇总或账本快照

### 6.8 补录老卡（P1，非本期）

- APP 内**不存在另一套「旧卡产品线」**；补录是指将**线下或未进系统的持卡**录入 **当前建卡体系**（选模板 / 权益 / 原办卡日等）。
- **本期不交付**；**P1 新增**。细则可对齐 `logic.html`：必填原办卡日，`卡到期日 = 原办卡日 + 模板有效期`（永久模板则到期为空）。
- 补录统计口径（是否计入办卡次数/总额）P1 立项时另定。

### 6.9 主路径

```text
列表 → 添加会员卡 → Step1 → Step2 → Step3 → 保存
     → 办卡 → 选会员 → 支付成功
     → 退卡/延期 → 确认
列表 → 下架 / 重新上架 / 克隆
详情 → 有持卡时编辑禁用 → 克隆改规则
```

### 6.10 验收要点（仅会员卡 · 本期）

- [ ] 三步可建组合卡并保存策略组合。
- [ ] 有持卡不可改模板规则；克隆后可改。
- [ ] 下架不可新办；已持卡可退/延。
- [ ] 延期只改卡到期日；含折扣延期费不计入办卡统计。
- [ ] 退卡建议金额与附录算例一致（至少算例 A）。
- [ ] 修改实际退款有审计日志；无部分退入口。
- [ ] 业绩轴可配置但不验收计绩结果。
- [ ] **不验收**价目管理、会员管理、补录老卡（补录属 P1）。

## 7. 规则附录（定稿）

> 权威：`logic.html` + `computeSuggestedRefund*`。界面副标题为简化句，**实现与验收以本附录为准**。

### 7.1 Q1 · 有效期

| 规则 | 定稿 |
|------|------|
| 卡级有效期 | 仅约束 **项目、折扣** |
| 面值 | **永久有效** |
| 宽限期 | **无** |
| 恢复 | 延期复活或重新办卡 |

### 7.2 Q2 · 延期

见 §6.6。

### 7.3 Q3 · 劳动业绩（产品目标 · 本期不实现运算）

- 字段 `performanceRule` 本期必存。
- 公式见 `logic.html` Q3；**不得写入本期验收**。
- 退卡只读 `refundRule`，与计绩不强制共用 runtime。

### 7.4 Q4 · 退卡公式

**建议退款（均值 / 减值共通）**

\[
建议退款 = \max(0,\ 办卡实付 - \sum 消耗价值)
\]

- Σ剩余价值可展示，**不参与封顶**。
- 消费顺序：余额先充值后赠送；项目先购买次后赠送次。

#### 名义价值池

| 权重 | 定义 |
|------|------|
| 余额名义 | 充值 + 赠送面值 |
| 项目权重 | 挂牌价 × (购次+赠次)；不限次不参与 |
| 折扣权重 | 挂牌价（或适用口径） |
| 名义合计 | Σ 权重 |

#### 均值 · 消耗价值

| 线 | 消耗价值 |
|----|----------|
| 余额 | 已用充值/赠送按分摊实付折算 |
| 项目 | 已用 × (分摊实付 ÷ 总次数) |
| 折扣 | 累计已享优惠 |

#### 减值 · 消耗价值

| 线 | 消耗价值 |
|----|----------|
| 余额 | 同均值 |
| 项目 | **挂牌价 × 已用**（办卡快照价） |
| 折扣 | 同均值 |

#### 锚定算例 A · 纯次卡

实付 500；洗剪吹 68；购 8 + 赠 2；已用购买 2。

| 模式 | 消耗 | 建议退 |
|------|------|--------|
| 均值 | 50 × 2 = 100 | **400** |
| 减值 | 68 × 2 = 136 | **364** |

更多算例见 `logic.html` B–E。

### 7.5 策略默认与冻结

- 默认两端均为 `alloc_pool`。
- 办卡后策略与挂牌价快照冻结；改模板/日后改价 **不回溯**已办卡。
