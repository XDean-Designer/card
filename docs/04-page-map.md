## 2. 前端状态要点

| 场景 | 表现 |
|------|------|
| `validCardCount > 0` | 详情编辑禁用；提示「已有持卡，不可编辑；请复制建卡后修改」 |
| `shelved` | 列表在「已下架」；不可新办；可重新上架 / 复制建卡 |
| Step1 有效期限 | 默认 **永久**；含不限次项目后不可再选永久 |
| Step2 权益开关 | 右侧 Switch；关闭仅收起展开区，**保留已配数据**（E2） |
| Step2 结果卡 | 白底撕口票券卡；标题=项目范围；副文=次数或折扣；整卡可编辑，右侧删除 |
| 项目范围文案 | 全选在售目录 →「全部项目」；否则全部项目名用 `、` 拼接（可换行，无省略号） |
| 添加权益确定后 | 回 Step2，**对应开关打开且展开区展开** |
| 添加项目权益 · 计次模式 | 「每项 / 共计」横向排列；右侧圆圈 `?`；点击弹窗展示当前状态说明（含不限次文案）；不限次时「共计」禁用但 `?` 仍可点 |
| 计次语义 | **每项**：各项目各自相同次数；**共计**：所选项目混用共享次数池（`sharedGroupId`）；退卡共计组权重用组内均价 |
| 次数设置文案 | 权益次数 / 赠送次数 / 不限次（原「购买次数」「次数模式」已改） |
| 不限次 × 永久 | 弹窗标题「设置有效期限」；内嵌日/月/年；**取消** / **确定**；确定写回卡有效期后保留不限次 |
| 选择项目 | 展示「已选 N 项」；至少 1 项才能确定；列表不展示价格 |
| Step3 分区标题 | 「消耗及退卡计算规则」「业绩计算规则」 |
| Step3 用卡策略 | 退卡：均值 / 减值；业绩：均值 / **原价**；纯折扣/不限次副标题写「总有效期」（无具体天数）；`?` 打开明细 |
| 永久卡 / 纯面值卡 | 退卡/延期列表仍展示「延期」；点击 toast「该卡永久有效，无需延期」，不展开面板 |
| 含折扣延期 | 展示费用，可改/0 |
| 退卡 Sheet | 剩余上、消耗下；策略角标均值/减值；文案「建议退款」 |
| 实退改写 | 允许；提交走审计 |
| 详情页布局 | 卡头独立；其下两大白底板块「运营数据」「卡面信息」外分、板内合并（细分割线）；字段分组顺序不变 |
| 已下架详情 | 顶栏「已下架」+「复制建卡」；底栏「重新上架 / 已持卡管理」 |
| 列表 / 卡面 | 「+复制建卡」（原克隆） |
| 步骤条间距 | Step2 步骤条与内容卡间距 **16px** |
