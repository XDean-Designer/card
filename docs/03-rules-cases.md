## 1. 有效期（Q1）

| 规则 | 定稿 |
|------|------|
| 卡级 `cardExpireAt` | 仅约束 **项目、折扣** |
| 面值 | **永久有效**，过期后仍可消费/参与退卡 |
| 宽限 | **无**；到期日当日项目/折扣硬失效 |
| 恢复 | 延期复活或重新办卡 |

消费校验：面值不校验卡到期；项目/折扣须未过期。

---

## 2. 延期（Q2）

| 项 | 规则 |
|----|------|
| 操作 | 仅 `cardExpireAt += N` 或延为永久 |
| 无折扣权益 | 免费，`feeAmount=0` |
| 有折扣权益 | 可收费；默认可：`折扣分摊购卡价 × 延期天数 ÷ 原有效期`；商家可改/0 |
| 流水 | 独立延期订单；**不计入办卡总额/次数** |
| 永久卡 | UI 仍展示延期入口；点击提示「永久卡无需延期」；接口拒绝（`EXTEND_NOT_ALLOWED`） |

---

## 3. 退卡总公式（Q4）

\[
建议退款 = \max(0,\ 办卡实付 - \sum 消耗价值)
\]

- Sheet 可展示 Σ剩余价值，**不得**用其封顶建议退。  
- 整卡退；确认后失效清零。  
- 消费顺序：余额先充值后赠送；项目先购买次后赠送次。  
- 策略读实例快照 `usagePolicySnapshot.refundRule`。

---

## 4. 名义价值池（组合卡共用）

| 权重 | 定义 |
|------|------|
| 余额名义 | 充值面值 + 赠送面值 |
| 项目权重 | 快照挂牌价 × (购次+赠次)；**不限次不计** |
| 折扣权重 | 快照挂牌价（或全部项目均价口径） |
| 名义合计 | Σ 权重；作分母 |

---

## 5. 均值模式 `alloc_pool`

| 线 | 消耗价值 |
|----|----------|
| 余额 | 已用充值×(余额分摊/充值面值) + 已用赠送×(余额分摊/总面值) |
| 项目 | 已用次数 × (该项目分摊实付 / 总次数) |
| 折扣 | 累计已享优惠 `savedTotal` |

余额分摊实付 = 办卡实付 × (余额名义 / 名义合计)。

---

## 6. 减值模式 `retail`

| 线 | 消耗价值 |
|----|----------|
| 余额 | **同均值** |
| 项目 | **快照挂牌价 × 已用次数** |
| 折扣 | **同均值**（已享优惠） |

---

## 7. 折扣档换算

- `tickIndex` ∈ [0, 20]；`20` = 原价  
- 折后价 = 原价 × `tickIndex / 20`（例：17 → 8.5 折）  
- `mode=fixed`：折后价 = `amount`  
- 已享优惠单次 = max(0, 原价 − 折后价)

---

## 8. 锚定算例 A（本期必过单测）

**输入**

- 实付 `paidAmount = 500`  
- 纯项目：洗剪吹挂牌 `68`；购 8 + 赠 2；已用购买 `2`，赠送未用  

**均值**

| 步骤 | 结果 |
|------|------|
| 名义合计 | 68 × 10 = 680 |
| 分摊实付 | 500 |
| 单次摊销 | 50 |
| 消耗 | 50 × 2 = **100** |
| 建议退 | 500 − 100 = **400** |

**减值**

| 步骤 | 结果 |
|------|------|
| 消耗 | 68 × 2 = **136** |
| 建议退 | 500 − 136 = **364** |
