先读这几条（贯穿全文）

  1. 新系统用 权益组合（面值 / 项目 / 折扣）建卡，不再选「五种卡类型」；整套取代旧会员卡系统。
  2. 创建固定 三步：基本信息 → 权益组合 → 用卡策略；Step2 至少开启并配置一类权益。
  3. 当前有效持卡数 > 0 时模板 不可编辑（可复制建卡、可下架、可办卡/退卡/延期）。
  4. 已下架不可新办；已持卡仍可退卡/延期；可重新上架或复制建卡。
  5. 退卡为 整卡退；建议退款可按策略计算，实际退款金额可改后确认。
  6. 用卡策略两轨独立：消耗及退卡计算规则、业绩计算规则，各可选均值或减值/原价。
  7. 接口路径/字段名以联调为准；本文给 能力与规则，标 【待联调确认】 处不臆造现网 URL。

交互行为与文案以 demo.html 当前表现 为准；本文把规则与验收写全，便于无历史上下文的机器与人直接开工。

模块 1　背景 / 目标 / 范围 / 能力映射

读者：前端 ✓ · 后端 ✓ · 测试 ✓

1.1 背景

原先的会员卡系统将卡划分为 5 种固定类型，每种类型套用 预设模板字段。商户创建会员卡时：

  • 只能在类型预设内填字段，自由度不足；
  • 每次都要面对 繁复的设置页 逐项填充，创建成本高。

本期交付的「会员卡管理」将 整套取代 上述旧系统（非长期并行两套产品线），对字段按能力重新解构。

1.2 目标

目标 说明
取代旧系统 新建模与流程成为唯一建卡体系
能力覆盖 可配置出与旧五类 功能等价 的卡，也可配置旧体系 无法做出 的组合卡
创建更简 三步完成：基本信息 → 权益组合 → 用卡策略
运营闭环 列表 / 详情 / 上下架 / 复制 / 办卡 / 延期 / 整卡退卡
策略可配置 退卡消耗与员工业绩规则在建卡时选定并随模板保存

1.3 本期做（In）

  • 会员卡模板：创建、编辑（无持卡时）、复制建卡、在售/已下架列表与详情
  • 权益组合：面值、项目（每项/共计、购买/赠送/不限次）、折扣（比例/固定金额）
  • 卡面主题色（5 色）、卡有效期限（日/月/年/永久，受不限次约束）
  • 用卡策略：消耗及退卡计算规则、业绩计算规则
  • 办卡（多选既存会员）、办卡成功
  • 已持卡：延期、整卡退卡（估值 + 实退可改）
  • 上下架及文案提示

1.4 本期不做（Out）

  • 开单结账全链路（选客、开单台、支付等）— 不在本文范围
  • 项目管理（价目表）作为独立产品交付（建卡时「选项目」依赖价目数据源，见依赖说明）
  • 会员管理（会员 CRUD；办卡仅从既存会员列表多选）
  • 部分退卡、劳动业绩独立运算台、预约联调
  • 原型工具栏「无数据/有数据」造数按钮 — 仅演示，不进正式产品

1.5 旧能力 ↔ 新组合权益（能力映射，不写废弃产品名）

旧体系常见能力（语义） 新体系如何配置
储值/充值类（买赠面值） 开启 面值：填面值金额；赠送 = 面值 − 购买金额（自动）
次卡/项目次数类 开启 项目：选适用项目；每项或共计；购买/赠送次数或不定次
折扣/会员价类 开启 折扣：选适用项目；标尺折扣或固定金额
期限卡 Step1 有效期限；含不限次项目时不可为「永久」
组合（储值+项目+折扣等） Step2 同时开启多类权益 并分别配置
旧类型无法覆盖的组合 任意权益开关组合 + 独立用卡策略

映射不足时 以新字段与原型行为为准，不编造旧系统字段表。

模块 2　用户场景与成功标准

读者：前端 ✓ · 后端 了解 · 测试 ✓

2.1 角色

角色 说明
店主（默认） 可进行本文全部操作
店员 【待产品确认】 本期原型未表现差异；权限见模块 9.2 占位

2.2 核心场景

场景 用户期望
首次建卡 三步填完保存 → 上架 → 可立即办卡
配置组合权益 面值/项目/折扣按需勾选配置，预览卡面
有持卡后改规则 不能直接编辑模板；复制建卡后改新卡
停售 下架后新客不能办；已持卡不受影响
再开售 重新上架后可再办
为会员办卡 选未持该卡的会员，可多选，办卡成功
延期 对非永久有效持卡设置延期时长（可含参考费用）
退卡 看到建议退款与明细，改实际退款后确认整卡退

2.3 成功标准（可判定）

  1. 空店可创建至少一张在售卡并出现在「在售」列表。
  2. Step2 未配置任何权益内容时 不能 进入 Step3 / 不能保存成功。
  3. 当前有效持卡 > 0 时详情「编辑」不可用，并提示复制建卡。
  4. 已下架卡列表无「办卡」入口；详情可「已持卡管理」（退卡/延期）。
  5. 办卡后持卡人数/办卡次数等运营数据按规则增加（见原型统计逻辑）。
  6. 退卡确认后该会员不再作为有效持卡；实退金额以确认值为准写入结果提示。

模块 4　状态机

读者：前端 ✓ · 后端 必看 · 测试 必看

4.1 模板上架状态

状态 产品表现 含义
在售 shelved=false 「在售」列表；可办卡；详情可下架/编辑（未锁时） 新客可办理
已下架 shelved=true 「已下架」列表；不可新办；详情可重新上架、复制、已持卡管理 停售 
在售 ──确认下架──► 已下架 ──确认上架──► 在售
  • 下架：不影响已持卡客户。
  • 已下架：选择会员 Sheet 仅「退卡/延期」（隐藏办卡 Tab）。

4.2 模板编辑锁定

条件 结果
当前有效持卡数 validCardCount == 0 可进入编辑三步并保存覆盖原模板
validCardCount > 0 不可编辑；详情「编辑」禁用；提示复制建卡 
无持卡 ──编辑──► 三步表单 ──保存修改──► 更新原模板 → 回详情
有持卡 ──点编辑──► 拦截（toast）──► 引导复制建卡 → 新建模板

4.3 表单模式 formMode

模式 入口 保存结果
create 「添加会员卡」/空态 新建在售模板 → 创建成功页
edit 详情「编辑」（未锁定） 更新原模板 → 详情；toast「已保存」
duplicate 「+复制建卡」 新建模板（名称默认「原名（副本）」）；原卡不变 → 创建成功页

复制模式三步均展示横幅：正在基于「{原卡名}」创建新卡，保存后原卡不受影响。

4.4 列表 Tab

listShelfTab: active | shelved，仅切换列表数据源与空态文案，不改模板状态。

4.5 持卡相关（实例层）

概念 说明
未持卡 办卡 Tab 可选
已持卡（有效） 退卡/延期 Tab；可退卡；非永久可延期
已失效 展示「已失效」；操作以原型表现为准
永久有效 延期入口提示无需延期

模块 5　交互细则 / 异常流

读者：前端 ✓ · 后端 了解 · 测试 ✓

5.1 总原则

  1. 列表点卡面进详情；footer 按钮不冒泡进详情。
  2. 创建/编辑/复制均为三步；Step1/2「下一步」、Step3「保存/保存修改」。
  3. Step2 必须至少有一类已开启且内容有效的权益。
  4. 金额类输入使用原型金额键盘交互（正式产品对齐现网数字键盘即可）。
  5. 所有拦截以 toast 或对话框给出 明确中文原因（见模块 9）。

5.2 离开与返回

  • 各步标题栏返回：回到上一步或列表（与原型一致：Step1 回列表，Step2→1，Step3→2）。
  • Sheet 点遮罩关闭：取消当前 sheet，不提交。
  • 【待联调确认】 三步中途系统返回是否弹「放弃编辑」：原型未做拦截时，正式产品可按现网统一规范补齐。

5.3 校验失败（须拦截前进/保存）

时机 条件 提示（原型文案）
Step1 下一步 卡名称为空 请填写卡名称
Step1 下一步 购买金额为空 请填写购买金额
Step1/相关 有效期限未设妥 请设置有效期限
Step2 下一步 未选任何权益内容 请至少选一项权益内容
Step2 开启面值但未填面值 请填写面值金额
Step2 开启项目但无已保存项目权益 请至少保存一个项目权益
Step2 开启折扣但无已保存折扣权益 请至少保存一个折扣权益
项目权益确定 未选项目 请先选择项目范围
项目权益确定 未设次数且非不限次 请设置次数或选择不限次
折扣确定 未选项目 请先选择折扣范围
折扣固定金额 ≤0 或空 请输入大于 0 的固定金额
折扣比例 未设置 请设置折扣
编辑锁定 有持卡仍保存/进编辑 已有持卡，不可编辑；请复制建卡后修改
不限次 + 永久 冲突 弹窗改有效期，或 toast：不限次项目须先设置卡有效期（不可为永久）
办卡确定 未选会员 请至少选择一位会员
延期 永久卡 该卡永久有效，无需延期
延期 未设时长 请设置延期时长
延期 参考费用为负 参考费用不能为负数
退卡 实退为空 请填写实际退款金额

5.4 空态

位置 文案
在售空 标题「暂无会员卡」；说明点击底部「添加会员卡」开始创建

6.1 会员卡列表

结构

  • 标题：会员卡管理
  • Tab：在售 | 已下架
  • 列表项：卡面条（VIP 标 + 卡名 + 价格）+ 权益摘要面板 + footer
  • 底栏：添加会员卡

在售 footer：+复制建卡、办卡/退卡
已下架 footer：仅 +复制建卡

卡面摘要面板

  • 面值：面值 ¥X · 赠送 ¥Y
  • 项目：项目名 + 次数摘要（10次 / 10次+送1次 / 不限次）
  • 折扣：项目名 + N 折 或 ¥金额
  • 期限：有项目/折扣展示卡有效期；纯面值展示「永久有效」；否则「—」

跳转

操作 结果
添加会员卡 formMode=create → Step1
点卡面 详情
+复制建卡 formMode=duplicate，预填，卡名「原名（副本）」→ Step1
办卡/退卡 打开选择会员 Sheet（办卡 Tab）

6.2 Step1 基本信息

字段 必填 说明
卡名称 ✓ 输入；placeholder「请输入」
购买金额 ✓ 金额；placeholder「请输入」
有效期限 ✓ 数量 + 单位：日 / 月 / 年 / 永久（默认永久）
卡面模板 ✓ 五色单选，默认品牌红；下方实时预览

步骤条文案：第 1/3 步 · 基本信息
底栏：下一步 → Step2

隐藏字段：适用对象 audience 默认 ['all']，原型 Step1 无 UI；后端可按现网对象模型扩展，【待联调确认】。

6.3 Step2 权益组合

三个独立 Switch 区块（可多开）：

6.3.1 面值

  • 描述：购买与赠送的面值，永久有效。
  • 面值金额（可编）
  • 赠送（只读）= 面值金额 − 购买金额（按原型计算与展示格式）

6.3.2 项目

  • 描述：包含、赠送项目的次数
  • 已保存权益以 ticket 展示；入口 +添加项目权益 → 项目权益页
  • ticket meta：每项/共计 + 次数摘要

6.3.3 折扣

  • 描述：比例或定额折扣
  • ticket + +添加折扣权益 → 折扣权益页

底栏：下一步 → Step3（通过校验后）

6.4 项目权益页

  • 标题：添加项目权益 / 编辑项目权益；右侧「管理项目表」（跳转价目依赖，【待联调确认】 是否本期可点）
  • 适用项目：列表 Switch；全选 / 反选
  • 计次模式：每项 | 共计
  • 每项：每个项目各自获得相同次数；不限次时各项目独立不限次
  • 共计：所选项目共享同一次数池；不限次时不可选共计
  • 权益次数 / 赠送次数 / 不限次
  • 页内「确定」写回 Step2 ticket

6.5 折扣权益页

  • 适用项目：同项目页选择交互
  • 打折（标尺：免费 ↔ 原价，展示如「8 折」）或 固定金额（输入 >0）
  • 「确定」写回 Step2

6.6 Step3 用卡策略

配置项 选项 默认
消耗及退卡计算规则 均值模式 / 减值模式 均值
业绩计算规则 均值模式 / 原价模式 均值

均值模式展示动态 hint（代入办卡实付、总权益面值）；? 打开明细说明。
底栏：保存 / 保存修改。

6.7 创建成功

  • 文案：会员卡已创建；已上架，可立即为会员办理
  • 展示新建卡预览
  • 按钮：返回列表 | 立即为会员办卡 → 选择会员（办卡）

6.8 会员卡详情

在售顶栏操作：下架、编辑（锁定时禁用）
已下架顶栏：已下架标记、复制建卡

运营数据（带「运营数据说明」）：

模块 8　业务规则与算例

读者：前端 ✓ · 后端 必看 · 测试 ✓

8.1 赠送面值

当开启面值权益：

[
赠送 = \max(0,\ 面值金额 - 购买金额)
]

（展示与存贮格式与原型一致，通常为金额字符串/两位小数。）

8.2 项目计次

模式 规则
每项 per_item 每个适用项目各自拥有相同的购买/赠送次数或各自不限次
共计 shared 多个项目共享一个次数池；消耗任一项目扣同一池
不限次 不可选共计；卡有效期 不能为永久

8.3 用卡策略公式（配置期文案）

消耗及退卡 · 均值 alloc_pool

  • 一般：消耗价值 = 项目面值 ×（办卡实付金额 ÷ 总权益面值）
  • 纯折扣 / 特殊卡型：按原型 hint 与明细弹窗（如按有效期天数比例等）实现，须与估值接口一致。

消耗及退卡 · 减值 retail

  • 消耗价值 = 项目面值（按已消耗权益的原价累计）

业绩 · 均值 alloc_pool

  • 员工业绩 = 项目面值 ×（办卡实付 ÷ 总权益面值）
  • 不限次项目可附加：劳动业绩按价目表原价计（原型 hint）

业绩 · 原价 retail

  • 员工业绩 = 项目面值

详情展示标签：均值退卡 / 减值退卡 · 均值计绩 / 减值计绩。

8.4 整卡退卡建议退款（均值池示意）

原型均值退卡逻辑概要：

  1. 取办卡实付 purchase
  2. 按账本累计已消耗价值 consumed（余额消耗、项目次数×分摊或原价、折扣已省等，细则与 refundRule 一致）
  3. 建议退款 = max(0, purchase - consumed)
  4. 界面展示分项明细；实际退款默认建议值，店主可改后提交

减值模式按「项目面值」口径累计消耗（computeSuggestedRefundRetail 语义）。

无分项可估时：提示按办卡实付协商退款。

8.5 办卡与持卡约束

  • 同一会员对同一模板：未退卡前不可再次新办（办卡列表排除已持卡）。
  • 已下架模板拒绝新办。

8.6 算例（测试可对照）

算例 A · 纯面值
购买 1000，面值 1000，赠送 0。未消费。均值/减值建议退款均接近 1000（无消耗）。

算例 B · 项目每项
购买 500；项目「剪发」购买 10 次，单次面值 50；已用 2 次。
- 减值：消耗 ≈ 2×50 = 100；建议退款 ≈ 400（若实付 500 且无其他权益）。
- 均值：消耗 = 已用部分对应分摊价值（实付/总面值×已用面值）；建议退款 = 实付 − 消耗。

算例 C · 不限次
不可永久有效；退卡消耗按策略与有效期/原型明细计算。

（联调时用接口返回值与上表口径对齐；若现网公式有常量系数，以联调纪要更新本节。）
